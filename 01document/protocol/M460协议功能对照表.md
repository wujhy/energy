# M460协议功能对照表

本文根据 `01document/protocol` 下 4 份协议文档和 `M460` 源码整理，目标是把“协议文件 -> 功能 -> M460实现 -> `energy` 承接”对应起来，便于后续整合和补齐。

整合后的原则已经调整：`energy` 不再通过原 M460 中间协议间接采集蓄电池，而是直接整合 M460 的底层采集和控制能力。`600节采集模块端` 是下行采集的主协议；`SH Modbus` 是上行 RS485 控制协议；`980节 PC 端` 和 `980节网络端` 不再作为 `energy <-> M460` 运行链路，只保留命令语义、状态机和实现方法供内部服务复用。

## 1. 协议文件总览

| 协议文件 | 主要定位 | 对应的 M460 功能面 | `energy` 承接方向 |
|---|---|---|---|
| `3_CM-02N 600节协议A0版_采集模块端_20231025.xlsx` | 下行采集协议 | 采集模块信息、单体模块内阻/均衡、模块地址、连接条电阻测试电压、校准参数 | 6 路下行 RS485 直接采集蓄电池，是第一优先级 |
| `3_CM-02N 980节协议B0版_PC端_V2.0_20260415.xlsx` | 原 PC 主控中间协议 | 设置系统状态、内阻测试、均衡、时间同步、设备信息、网络参数 | 整合后不再作为中间通讯协议，仅保留命令语义和实现方法 |
| `3_CM-02N 980节协议B0版_网络端_V2.0_20250819.xlsx` | 原网络端中间协议 | TCP/JSON 上报、云端参数、MAC/IP、网络侧控制面 | 整合后不再作为 M460 网络中间协议；原 `energy` JSON/TCP 上报保留 |
| `SH 980节协议A0版_Modbus_V4.8_20260105.xlsx` | Modbus 寄存器协议 | 寄存器映射、读写控制、远程状态/命令 | 2 路上行 RS485 Modbus 控制协议 |

## 2. 采集模块端协议

### 对应的 M460 代码位置

- [rs485.c](../../02sourceCode/M460_Series_BSP_CMSIS_V3.00.002/SampleCode/NuMaker_M467HJ/LwIP_TCP_EchoClient/MyFiles/rs485.c)

### 主要功能

- 下行轮询和接收采集模块端帧
- 读取采集模块信息、连接条电阻测试电压等 600 节模块端数据
- 执行单体模块内阻测试、均衡、地址设置、内阻系数、校准参数等模块端控制
- 处理采集模块主动上报帧
- 电池组配置、组实时数据、告警、故障等 980 聚合视图由 `energy` 后续基于标准模型计算或映射生成，不直接等同于 600 节原始命令

### 600 节协议命令

| 请求/响应 | 功能 | M460侧含义 |
|---|---|---|
| `01/81` | 采集模块信息 | 模块基础信息 / 型号 / 版本 |
| `02/82` | 单电池监测模块内阻测试 | 注意：在 600 节协议中不是电池组实时数据 |
| `03/83` | 单电池监测模块均衡 | 单体模块均衡控制 |
| `08/88` | 设置模块地址 | 模块地址设置应答 |
| `76/F6` | 设置校准参数 | 工厂校准应答 |
| `18/A8` | 自动设置模块地址 | 自动编号/设置模块地址 |
| `0A` | 清除电池组单体调试数据 | 文档未给出明确响应码 |
| `0F` | 连接条电阻测试 | 文档未给出明确响应码 |
| `11/91` | 获取连接条电阻测试电压 | 上传单体电池电压、自动编号测试电压 |
| `12/92` | 设置内阻系数 | 设置内阻系数应答，地址 `0x00` 广播不回复 |

### 需要避免混淆的 980 节聚合命令

以下命令来自 PC 端/网络端 980 节协议，不属于 600 节采集模块端协议；整合后只能作为 `energy` 内部聚合输出或兼容实现参考。

| 请求/响应 | 980 节功能 | `energy` 新处理原则 |
|---|---|---|
| `02/82` | 获取/上传电池组信息 | 根据下行单体采集结果生成电池组信息 |
| `06/86` | 读取/上传电池组配置信息 | 映射到内部配置模型 |
| `07/87` | 获取/上传电池组报警状态 | 映射到标准告警模型 |
| `0B/8B` | 读取/上传电池组参数 | 映射到本地参数模型 |
| `0D/8D` | 获取/上传设备故障类告警状态 | 映射到标准故障模型 |
| `0E/8E` | 获取/上传设备型号及软件版本号 | 映射到设备信息模型 |
| `39/E9` | 获取/响应屏蔽报警参数 | 作为内部控制/配置查询参考 |
| `3B/EB` | 获取/响应自动编号、连接条电阻、内阻测试状态 | 作为内部测试状态查询参考 |

### M460 代码中的主链路

- `RS485_Command_Run()`：RS485 主循环
- `RS485_Read_Command_Package()`：读上位/串口命令帧
- `RS485_Command_Execute()`：命令执行分发
- `Protocol_Package_Run()`：状态机主循环
- `Battery_Communication_Function_Process()`：电池协议收发

### `energy` 承接方向

- 已落地为 `com.shanhe.project.collector.battery`
- 6 路下行 RS485 采集蓄电池，优先完成真实串口联调
- 以协议字段为准完成单体数据、组数据、告警、故障、参数解析
- 建立 `采集帧 -> 标准对象 -> 入库 -> 组信息计算` 的闭环
- 600 节模块端原始帧不再默认桥接旧 `BatteryHandler`
- 当前代码已将 600 节模块端自动轮询收敛为 `BatteryDeviceProtocolCode.MODULE_INFO(01/81)`，980 节 PC/网络端聚合命令收敛到 `BatteryAggregateCommandDefinition`，后续不得再混用。
- 当前已确认 M460 常规采集循环默认只轮询 `01/81`；`11/91` 属于连接条电阻测试流程，不进入默认自动轮询。
- 当前代码已将模块帧分发收敛到 `BatteryModuleFrameDispatcher` 和 `BatteryModuleFrameConsumer`，后续入库、计算、980 兼容输出都应作为显式 consumer 实现。
- 当前代码已阻断 980 聚合命令兼容入口直接下发到 600 节模块端通道，后续控制动作必须先落到明确的模块端控制映射。
- 当前代码已将 600 节 `81` 标准对象接入实时表，并可选触发电池组计算；`realtimeDataEnabled`、`groupCalculationEnabled` 默认关闭，避免未联调时影响现网采集。

### 主要功能

- PC 作为主节点，采集主机作为从节点
- 下发设置系统状态、测试、均衡、时间同步、地址修改等命令
- 查询设备信息、内阻测试配置、连接条电阻配置、MAC/IP/云端参数
- 配合状态机控制，进入 `BATTERY_EXECUTE_COMMAND` 后再返回 `BATTERY_EXECUTE_READ`

### 主要命令面

| 功能类 | 典型命令 | M460侧含义 |
|---|---|---|
| 系统状态设置 | `SERVER_SET_SYSTEM_STATE` | 进入测试、均衡、充电、备电等状态 |
| 内阻测试 | `SYS_SINGLE_INTERNAL_RESISTANCE_TEST`、`SYS_CONNECT_RESISTANCE_TEST` | 单体内阻 / 连接条电阻测试 |
| 电池均衡 | `SYS_BATTERY_EQUALIZATION` | 手动均衡、自动均衡相关控制 |
| 时间同步 | `SYS_UPDATE_TIME_ALL` | 统一更新时间日期 |
| 设备信息 | `SYS_DEVICE_TYPE_VER`、`SYS_GET_MAC_ADDRESS` | 型号、版本、MAC 查询 |
| 网络配置 | `SYS_SET_DEVICE_IP_ADDRESS`、`SYS_SET_CLOUD_SERVER_IP_ADDRESS`、`SYS_SET_SERVER_CLIENT_MODE` | IP/云端/工作模式配置 |
| 参数设置 | `SYS_SETTING_INTERNAL_RESISTANCE_COEFFICIENT`、`SYS_SET_SWOLLEN_VOLTAGE_REFERENCE` | 内阻系数、鼓包阈值等参数设置 |
| 屏蔽/告警 | `SYS_DISABLE_WARNING_CONFIGURE`、`SYS_ALARM_RELEASE` | 告警屏蔽、解除告警 |

### `energy` 承接方向

- 整合后不再保留 `energy <-> M460` 的 PC 端中间通讯链路
- 协议中的设置、测试、均衡、时间同步等命令语义可保留为 `energy` 内部命令服务
- 内部命令服务后续由上行 Modbus 写寄存器、页面操作或网络控制触发
- 当前不并入下行自动轮询

- 网络端与采集主机进行 JSON/TCP 上报或控制
- 处理设备 IP、云服务器 IP、客户端模式、MAC 地址读取
- 与 PC 端协议大量重叠，但通信载体转为网络

### 关键判断

- 这一层不是新增业务协议，而是把 PC 端 / 设备控制能力通过网络暴露出来
- `energy` 侧需要保留现有 JSON 上报链路，同时补齐网络控制场景

### `energy` 承接方向

- 不恢复 `energy <-> M460` 的网络端中间协议
- 保留原 `energy` JSON/TCP 网络上报能力
- JSON/TCP 的数据来源应切换为整合后的标准数据模型
- 如后续需要网络控制，应调用内部命令服务，而不是重新引入 M460 网络端协议链路

- 作为 2 路上行 RS485 Modbus RTU 控制入口
- 后续再扩展 Modbus TCP
- 先把寄存器读映射到标准数据模型
- 再把寄存器写动作映射到内部命令服务

## 6. 功能边界结论

1. `600节采集模块端` 对应的是 M460 的下行电池采集功能，整合后作为 `energy` 的 6 路下行 RS485 主协议。
2. `980节PC端` 对应的是 M460 的设备控制、测试、设置功能，整合后只保留实现方法，不再作为中间通讯协议。
3. `980节网络端` 对应的是 M460 的网络化控制和上报功能，整合后不再作为 M460 网络中间协议；原 `energy` JSON/TCP 上报继续保留。
4. `SH Modbus` 对应的是 M460 的第三方系统接入和寄存器控制功能，整合后作为 `energy` 的 2 路上行 RS485 Modbus 控制协议。

## 7. 对 `energy` 的实际落点

- 下行采集：6 路 RS485 直连蓄电池采集模块/电池组
- 数据处理：在 `energy` 内部完成单体数据入库、电池组信息计算、告警故障标准化
- 上行控制：2 路 RS485 Modbus RTU
- 网络上报：保留原 `energy` JSON/TCP
- 旧 `M460` 中间层协议：退出运行链路，收敛为内部方法和源码参考
